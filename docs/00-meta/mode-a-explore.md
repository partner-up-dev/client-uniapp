# Mode A: Explore

## Role

Use when key unknowns remain and the solution space must be mapped before safe solidification or execution.

Explore is a working posture, not a durable owner. It can appear inside any input route.

## Forbidden

- Do not treat exploration notes as durable truth.
- Do not edit PRD, TDD, deployment docs, or production code when uncertainty is still high.
- Do not let open-ended searching replace a concrete next question.

## Read-do steps

1. State the current unknowns and why they block progress.
2. Record temporary assumptions explicitly in `tasks/`.
3. Read only the smallest useful references.
4. Reduce unknowns into candidate decisions, claims, contracts, or hypotheses.
5. Decide whether to stay in Explore, move to Solidify, or switch to Diagnose.

## Exit criteria

- The next decision is framed clearly enough for Solidify or Execute.
- Remaining uncertainty is explicit.
