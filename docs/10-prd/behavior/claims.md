# Claims

## Browse partner requests

- Claim intent: users can discover partner requests through browse surfaces such as Explore and Home.
- Evaluation dimensions: relevant requests are visible, request summaries are understandable, navigation to detail is available.
- Evidence expectation: page flows and route mappings expose browse and detail surfaces.
- Source rationale: [_drivers/market-and-user-pressures.md](../_drivers/market-and-user-pressures.md), [_drivers/business-and-service-objectives.md](../_drivers/business-and-service-objectives.md)
- Realization pointers: [20-product-tdd/claim-realization-matrix.md](../../20-product-tdd/claim-realization-matrix.md)
- Impact on existing claims: supports application and communication claims.

## Create partner requests

- Claim intent: users can create partner requests with structured steps.
- Evaluation dimensions: request type, route/detail, and final submission intent can be captured.
- Evidence expectation: creation surfaces exist and preserve structured request data.
- Source rationale: [_drivers/business-and-service-objectives.md](../_drivers/business-and-service-objectives.md), [_drivers/hard-constraints.md](../_drivers/hard-constraints.md)
- Realization pointers: [20-product-tdd/claim-realization-matrix.md](../../20-product-tdd/claim-realization-matrix.md)
- Impact on existing claims: creates supply for browse and application flows.

## Apply to partner requests

- Claim intent: users can request to join partner requests and specify intended roles.
- Evaluation dimensions: applications can represent one or more role-specific sub-applications.
- Evidence expectation: application models and UI can capture role selection and application reasons.
- Source rationale: [_drivers/market-and-user-pressures.md](../_drivers/market-and-user-pressures.md), [_drivers/business-and-service-objectives.md](../_drivers/business-and-service-objectives.md)
- Realization pointers: [20-product-tdd/claim-realization-matrix.md](../../20-product-tdd/claim-realization-matrix.md)
- Impact on existing claims: depends on partner request detail and role semantics.

## Chat with partners

- Claim intent: users can coordinate through chat after partner-related activity.
- Evaluation dimensions: messages are visible, new messages update local activity state, chat can be entered from relevant surfaces.
- Evidence expectation: chat page, communication models, and active unread state support the flow.
- Source rationale: [_drivers/market-and-user-pressures.md](../_drivers/market-and-user-pressures.md), [_drivers/business-and-service-objectives.md](../_drivers/business-and-service-objectives.md)
- Realization pointers: [20-product-tdd/claim-realization-matrix.md](../../20-product-tdd/claim-realization-matrix.md)
- Impact on existing claims: supports active participation after browsing or joining.

## View notifications

- Claim intent: users can see surfaced activity for partner and chat events.
- Evaluation dimensions: notification surface exists and unread/activity state can be surfaced.
- Evidence expectation: notification page and communication state support visible updates.
- Source rationale: [_drivers/business-and-service-objectives.md](../_drivers/business-and-service-objectives.md)
- Realization pointers: [20-product-tdd/claim-realization-matrix.md](../../20-product-tdd/claim-realization-matrix.md)
- Impact on existing claims: supports chat and partner request follow-up.

## Manage profile

- Claim intent: users can view and maintain account/profile details used across partner flows.
- Evaluation dimensions: profile information can be displayed and edited where supported.
- Evidence expectation: account/profile surfaces and account state exist.
- Source rationale: [_drivers/business-and-service-objectives.md](../_drivers/business-and-service-objectives.md), [_drivers/hard-constraints.md](../_drivers/hard-constraints.md)
- Realization pointers: [20-product-tdd/claim-realization-matrix.md](../../20-product-tdd/claim-realization-matrix.md)
- Impact on existing claims: supports identity continuity across partner interactions.
