# Workflows

## Launch and login

- App launches and attempts account login
- User proceeds with session state if available

## Browse and view partner requests

- User opens explore or home
- User navigates to a partner request detail

## Apply to a partner request

- User submits a partner application
- Application contains one or more role sub-applications
- Approval of any sub-application marks the application as approved

## Create a partner request

- User starts a creation flow
- User completes step-by-step inputs
- Request becomes available for partners

## Chat and notifications

- User opens chat from a partner request or notification
- Messages update local unread state
- Notifications surface new activity

## Profile management

- User views and edits profile details
- Changes propagate to partner flows
