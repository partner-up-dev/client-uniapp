# Me 页面

## 概述

"我的"页面是用户个人中心的主入口，展示用户基本信息并提供各类账户相关操作的入口。该页面支持登录和未登录两种状态。

## 页面位置

- 路径: `pages/me/me`
- 类型: TabBar 页面 (第4个标签)
- 模块: `account`

## 功能特性

### 0. 状态切换

页面根据用户登录状态自动切换显示内容：

- **未登录状态**: 显示欢迎信息和登录按钮
- **已登录状态**: 显示完整的用户信息和操作入口

### 1. 未登录状态

- **欢迎文字**: 显示"欢迎您"
- **默认壁纸**: 展示预设的壁纸图片
- **登录按钮**: 左下角的绿色登录按钮，点击触发登录流程

### 2. 已登录状态 - 用户信息展示

- **背景壁纸**: 展示用户设置的个人壁纸图片，若未设置则显示渐变背景
- **头像**: 大尺寸圆形头像展示
- **用户昵称**: 显示用户的昵称
- **加入时间**: 显示用户注册/加入平台的日期

### 3. 已登录状态 - 主要功能入口

#### 我的搭子请求

- 展示入口: Cell 组件
- 功能说明: 查看"我发布的、参与的、收藏的"搭子请求
- 交互: 点击跳转到搭子请求列表页

#### 操作按钮组

- **编辑资料**: 跳转到个人资料编辑页面
- **登出**: 执行账户登出操作
- **查看**: 查看更新或其他信息（待定）

## 数据模型

### 核心数据

使用 `business/account/base.ts` 的 `AccountBaseProfile` 类:

```typescript
{
  id: string;
  created_at: Date;
  nickname: string;
  bio: string | null;
  wallpaper: string | null;
  avatar: string | null;
  age_range: number | null;
  gender: string | null;
  mbti: string | null;
}
```

## 组件复用

- `common/safeAreaInset`: 处理顶部状态栏、底部安全区和 TabBar 占位
- `common/avatar`: 展示用户头像，支持大尺寸
- `common/cell`: 展示"我的搭子请求"入口
- `common/PUButton`: 操作按钮组

### 设计要点

1. **壁纸区域**: 360px 高度，支持图片或渐变背景
2. **用户卡片**: 绝对定位在壁纸右下角，带有文字阴影增强可读性
3. **内容区域**: 24px 上下内边距，16px 左右内边距
4. **按钮组**: 底部对齐，每个按钮间距 12px
