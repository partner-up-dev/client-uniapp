# PRRoute

## Rationale

将页面中的路线展示视图抽离为可复用组件，降低页面复杂度并提升复用性。

## Goals

- 展示搭子请求的路线摘要（地图预览、时间与类型、路线点列表）。
- 仅依赖一项输入 `route`。

## Specification

- 输入：`route: Route`。
- 输出：无；纯展示组件。
- 结构：地图预览、日期与类型、路线项列表。
- 样式：使用设计代币，独立于页面样式。

## Implementation

组件包含模板、脚本与样式三部分，遵循组件开发规范。

### Props

| 属性名 | 类型  | 默认值 | 必填 | 说明           |
| ------ | ----- | ------ | ---- | -------------- |
| route  | Route | -      | 是   | 路线业务模型。 |
